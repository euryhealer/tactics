1. Vision

The economy should support normal farming progression while preparing for real player-to-player trading. Gold remains the main gameplay currency. Premium tokens are a separate constrained currency for rare items, special animals, cosmetics, limited listings, boosts, and future monetization-safe mechanics.

2. Currencies

Currency Purpose Current status
currencies.gold Normal game economy: seeds, land expansion and direct produce sales. Implemented as profile state.
currencies.premiumToken Premium/rare economy: player marketplace purchases, special saplings, animals, future cosmetics and listing slots. Implemented as profile state.

6. Player Marketplace

The first marketplace is local/mock only. It uses static listings stored in the player profile and removes a listing after purchase. Player marketplace listings use premium token only. This creates the UI and data shape before backend integration. 

{
  "id": "mock-apple",
  "seller": "Orchard Guild",
  "itemId": "apple",
  "qty": 8,
  "priceCurrency": "premiumToken",
  "unitPrice": 1
}

7. Backend Ownership

Once this becomes real player trading, the backend must own currency balances, inventory quantities, listing creation, purchase validation, and transaction logs. The client can preview transactions, but it should not be authoritative.
